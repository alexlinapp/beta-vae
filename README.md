# Beta-VAE

## Overview
Implementation of $\beta$-VAE and $\beta$-TCVAE on dSprites. Comparison between different values of $\beta$ and KL-Annealing

## Theory

#### Original VAE objective
The original VAE objective is to **maximize** the **Evidence Lower Bound** (ELBO) $\mathcal{L}(\theta,\phi;\mathbf{x},\mathbf{z})$:
$$
\log p_\theta(\mathbf{x}|\mathbf{z})
\geq
\mathcal{L}(\theta,\phi;\mathbf{x},\mathbf{z})
=
\underbrace{\mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x})}
\left[\log p_\theta(\mathbf{x}|\mathbf{z})\right]}_{\text{reconstruction accuracy}}
-
\underbrace{D_{KL}\left(q_\phi(\mathbf{z}|\mathbf{x}) \,\|\, p(\mathbf{z})\right)}_{\text{``closeness'' of } q(\mathbf{z}|\theta) \text{ to } p(\mathbf{z})}
$$
where we typically assume that $p(\mathbf{z}) \sim \mathcal{N}(0, I)$.

Hence there is a tradeoff between accurately reconstructing the image and latent space regularization. We need to encode enough information to reconstruct the input while keeping the latent distribution **close** to a simple prior so new samples can be easily generated by drawing $\mathbf{z} \sim \mathcal{N}(0, I)$. 

#### Beta VAE Objective
The idea is to impose a regularization factor on the $D_{KL}$ term above, thus constraining the model and discouraging it from learning ``messy'' latent distributions, i.e. encourage the latent distribution to be close to the prior. Recall that $D_{KL}(p(x) || q(z)) = 0$ iff $p = q$ almost eveywhere (not equal on a set of measure zero). 

By increasing the $D_{KL}$ weight with $\beta$ (hyperparameter), this creates a stronger information bottleneck. The latent dimension should only preserve the most important factors of variation needed. Since $p(\mathbf{z})$ is factorized, we should expect that by placing more emphasis on $q(\mathbf{z} \mid \theta)$ to be close to $p(\mathbf{z})$, $q$ itself should be factorized as well, thus the latent dimensions are independent and ideally, represent independent factors. 

**Cavaet**: Disentanglement is not guarenteed. Further, stronger regularization on the $D_{KL}$ term forces lower reconstruction quality and we lose information about the input $\mathbf{x}$ as we move closer to the prior of $p(\mathbf{z}) \sim \mathcal{N}(0, I)$. Recall that good reconstruction requires that $q_{\phi}(\mathbf{z} | \mathbf{x})$ place high probability on latent codes for which $p_{\theta}(\mathbf{x} | \mathbf{z})$ is high. But forcing $p(\mathbf{z})$ to follow the standard normal can possible prevent this. This is also why when you set $\beta = 100$ (or absurdly high) you just see a giant blob of white on the dSprites dataset. We are basically fully normal.

## Dataset
- dSprites
- ground-truth factors:
  - shape
  - scale
  - orientation
  - x-position
  - y-position

## Implementation
- Basic MLP Encoder and Decoder Architecture
- Latent Dimension of 10
- optimizer, both AdamW and AdaGrad used
- loss function: negative log likelihood of multvariate bernoulli distribution with conditionally independent pixels
- Linear KL Annealing

## Experiments
- Original VAE: $\beta = 1$, no KL Annealing
    - Bad Scale: Sizing of reconstruction term takes average of pixels (64 x 64) rather than by image. Leads to higher emphasis of KL divergence, poor reconstruction
- Weighted 
- Weighted VAE: $\beta = 4$
- $\beta$ = 4 with linear KL annealing,

## Issues
- Very very sensitive to training dynamics (batch size and some sizing issues between KL divergence and reconstruction loss)
    - $\beta$-TCVAE is extremely sensitive to batch size (most likely due to overuse of monte carlo estimation and dataset size)

## How to Run

## Dataset

By default, datasets should be stored in:

```bash
datasets/
```

You can override this with:

```bash
python main.py --datasets-dir path/to/datasets
```

## Run a Basic Beta-VAE

Run the default configuration:

```bash
python main.py
```

Run a basic Beta-VAE experiment explicitly:

```bash
python main.py \
  --objective B \
  --beta 4.0 \
  --latent-dim 10 \
  --epochs 130 \
  --batch-size 128 \
  --lr 0.01 \
  --optimizer Adagrad \
  --likelihood bernoulli \
  --device cuda \
  --run-name basic_beta_vae
```

## Run on CPU

```bash
python main.py --device cpu --run-name cpu_test
```

## Use Adam Instead of Adagrad

```bash
python main.py --optimizer Adam --lr 0.001 --run-name adam_beta_vae
```

## Change the Latent Dimension

```bash
python main.py \
  --latent-dim 32 \
  --beta 4.0 \
  --run-name latent32_beta4
```

## Try Different Objectives

Burgess-style Beta-VAE:

```bash
python main.py \
  --objective B \
  --beta 4.0 \
  --run-name burgess_beta_vae
```

Higgins-style Beta-VAE:

```bash
python main.py \
  --objective H \
  --beta 4.0 \
  --run-name higgins_beta_vae
```

Total Correlation Beta-VAE:

```bash
python main.py \
  --objective T \
  --beta 4.0 \
  --run-name tc_beta_vae
```

## Beta Annealing

Run with linear beta annealing:

```bash
python main.py \
  --objective B \
  --beta 4.0 \
  --beta-annealing Linear \
  --beta-warmup 10000 \
  --run-name beta_annealing_test
```

Disable beta annealing:

```bash
python main.py \
  --beta-annealing None \
  --run-name no_beta_annealing
```

## Checkpoints

Checkpoints are saved every `checkpoint_freq` epochs.

Example:

```bash
python main.py \
  --checkpoint-freq 5 \
  --run-name checkpoint_test
```

## Run Directory

By default, outputs are saved under:

```bash
runs/
```

Example:

```bash
runs/basic_beta_vae/
```

You can change the output directory:

```bash
python main.py \
  --run-dir my_runs \
  --run-name experiment_1
```

By default, the run directory may include a timestamp. To disable timestamps:

```bash
python main.py \
  --run-name no_timestamp_test \
  --run-timestamp false
```

## Reproducibility

Set the random seed:

```bash
python main.py --seed 0 --run-name seed0_test
```

## Command-Line Arguments

| Argument | Default | Description |
|---|---:|---|
| `--latent-dim` | `10` | Size of the latent vector |
| `--alpha` | `1.0` | Alpha coefficient |
| `--beta` | `4.0` | Beta coefficient |
| `--gamma` | `1.0` | Gamma coefficient |
| `--lr` | `0.01` | Learning rate |
| `--epochs` | `130` | Number of training epochs |
| `--device` | `cuda` | Training device, usually `cuda` or `cpu` |
| `--objective` | `B` | Objective type: `B`, `H`, or `T` |
| `--beta-warmup` | `10000` | Warmup steps for beta annealing |
| `--train-split` | `0.8` | Fraction of dataset used for training |
| `--batch-size` | `128` | Batch size |
| `--run-name` | `default_run` | Name of the experiment run |
| `--run-dir` | `runs` | Directory for experiment outputs |
| `--checkpoint-freq` | `5` | Save a checkpoint every N epochs |
| `--datasets-dir` | `datasets` | Dataset directory |
| `--seed` | `0` | Random seed |
| `--likelihood` | `bernoulli` | Reconstruction likelihood |
| `--optimizer` | `Adagrad` | Optimizer name |
| `--beta-annealing` | `None` | Beta annealing schedule: `None` or `Linear` |
| `--run-timestamp` | `true` | Whether to add a timestamp to the run directory |
